ATLAS on Debian Squeeze (AMD64)

Posted on November 30, 2010 by Taro

Installing from Debian Repository

$ sudo aptitude install libatlas3gf-base
$ sudo aptitude install libatlas-base-dev

The second line is necessary only for development.

Installing from Source

Disable CPU throttling.  Find out how many CPUs I have:

$ cat /proc/cpuinfo
... info for each CPU ...

Check which CPU I have.  Mine is ﻿﻿Intel(R) Core(TM) i5 CPU M 520 @ 2.40GHz and there are four CPUs (really two CPUs but becomes four with hyperthreading).  Set each CPU’s governor as performance:

$ for i in {0..3}; do sudo cpufreq-set -g performance -c $i ; done

(Note “{0..3}” should reflect how many CPUs there are; for a dual core machine, it probably should be {0..1}.)

In case, do this first:

$ sudo aptitude install gcc gfortran

I download tarballs for ATLAS and LAPACK from the official repositories.  Here I use ATLAS 3.9.32 (still a development version) and LAPACK 3.3.0.  I assume the compiled library and header files will be installed at /usr/local/atlas-3.9.32, which will be symbolic linked at /usr/local/atlas.

Prepare the source directory:

$ mkdir tmp
$ cd tmp
$ wget http://www.netlib.org/lapack/lapack.tgz  # at the time of writing this is v3.3.0
... download the tarball for ATLAS from sourceforge ...
$ tar -xvjf atlas3.9.32.tar.bz2
$ mv ATLAS atlas-3.9.32
$ ln -s atlas-3.9.32 atlas
$ mkdir atlas/bld
$ cd atlas/bld
$ ../configure -b 64 --shared \
  --prefix=/usr/local/atlas-3.9.32 --with-netlib-lapack-tarfile=../../lapack.tgz

Hidden symbol in gcc error.  Now this may only be relevant to my current configuration, but when I linked some other program against my custom ATLAS build (I was compiling SExtractor), I have the following error:

/usr/bin/ld: prog: hidden symbol `__powidf2' in /usr/lib/gcc/x86_64-.../libgcc.a(_powidf2.o) is referenced by DSO
/usr/bin/ld: final link failed: Nonrepresentable section on output

which prevented me to build programs which link to ATLAS.  It took a while to find a fix, but to prevent this error, I need to modify lib/Makefile in the bld directory (after I run ../configure once) so that

liblapack.so : liblapack.a
        ld $(LDFLAGS) -shared -soname $(LIBINSTdir)/liblapack.so \
            -o liblapack.so  -rpath-link $(LIBINSTdir) --whole-archive \
           liblapack.a --no-whole-archive $(F77SYSLIB) -lgcc

“-lgcc” (in red) is added.

Specifying Architecture.  The build process might take very long time if I don’t specify right architecture.  Do this:

$ make xprint_enums ; ./xprint_enums

and I get a list of enums for different CPU architectures.  Mine is Intel Core i5, which isn’t actually listed but I figured it might be close enough to Core i7, so I chose that one (18).  I will feed this to the configure script.  Earlier I also found out that my CPU runs at 2.4 GHz.  I will also feed this info to the configure script as seen next.

Now we do:

$ ../configure -A 18 -b 64 -D c -DPentiumCPS=2400 -Fa alg -fPIC --shared \
  --prefix=/usr/local/atlas-3.9.32 --with-netlib-lapack-tarfile=../../lapack.tgz
$ time make build

(Note the options “-A 18″ and “-D c -DPentiumCPS=2400″ have been added.)  This takes about 15 minutes on my Lenovo T410s.

If I wish to do some checking, do:

$ make check
...
$ make time
...

If I don’t find any problem, finally do:

$ sudo make install
$ sudo ln -s /usr/local/atlas-3.9.32 /usr/local/atlas

That’s it.
